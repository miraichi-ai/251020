1Java集中学習：試験突破までの道

結論

AIなし、教科書のみで、毎日12時間×365日の集中学習。基礎本の徹底反復が試験突破を実現

背景

SIer企業の面接が決定。Java試験が実施されるまでの間に、集中学習開始。

  • C言語は習得済み、Javaは未経験
  • 試験までの限定期間に集中
  • AIツール（ChatGPT等）がない時代
  • インターネット情報も限定的

具体的な学習方法

「スッキリわかるJava入門」シリーズ

基礎本を徹底的に反復。毎日読んだため、本が真っ二つに折れ、2冊購入。

当時は第1弾。現在は第4弾まで続く有名シリーズ。

学習量と継続力

1年間の学習量：約12時間/日 × 365日

毎日休みなく学習。「もう学校に来ちゃったからやるしかない」という腹決めが、継続を支えた。

困難な点

Windows環境での環境構築が極めて困難

  • WSL2が存在しない時代
  • Windows OS上でLinuxを動かす必要があった
  • 1日2日かけてもエラーが解消できない
  • 10時間以上エラーと格闘することもザラ
  • Linuxの情報は豊富だが、Windowsの情報は少ない

環境設定の工夫

Linux OS とWindows OS を並列で動かせるようになったのは後年のこと。当時は「WindowsはIT学習の罠」だった。しかし、その困難な環境での学習が、後々のトラブルシューティング力を養った。

結論

基礎本の反復と、環境設定での試行錯誤が、実践的なスキル習得を実現

2Udemy・SQL・Git：プロ環境に必要な周辺スキル

結論

書籍学習から実務へのギャップを埋めるのが、オンライン講座とGitスキル習得

学習内容

Udemyオンライン講座

山浦さんのAWS・Linux・Git講座。当時は2015年の黎明期で講座も少なかった。

SQL習得

データベース操作の重要性。書籍に全てのリンクが付属しており、非常に有益。

Git習得

チーム開発に必須。「Gitをプッシュしてゴミを送ってくる」と怒られるのは常。

Python基礎

「入りシリーズ」のPython本。会社補助で購入。基礎言語として習得。

重要な気づき

Gitの重要性：

IT企業ではGitなしでは厳しい。Gitを知らないと新人も先輩も関係なく通用しない。

オンライン講座の時代性：

当時のUdemy講座は今見ると古いが、当時は最先端。情報不足の時代だったからこそ、オンライン講座の価値は大きかった。

結論

書籍 + オンライン講座 + Git習得で、プロエンジニアの下地が完成

3最初の企業：技術と現実のギャップ（1年4ヶ月）

結論

死ぬほど勉強したが、技術スキル ≠ 現場での活躍。コミュニケーションスキルの不足が露呈

背景と採用

基礎本のみでJava試験に合格。入社前課題でECアプリを作成。

  • 入社前課題：ECアプリ作成、SQL課題など実施
  • E-Commerceの企業。Javaでサイトを開発
  • 初めての現場環境

具体的な業務内容

技術スタック： Java、JavaScript、SQL

開発手法： ウォーターフォール開発

業務内容：

  • Excelで要件定義
  • 画面設計書作成
  • テストケース作成
  • ECサイトの運用・保守

直面した困難

技術はあるが、現場では全く活躍できなかった

  • 何をしたらいいのか全く分からないまま業務開始
  • コミュニケーション不足で状況把握ができない
  • 営業経験が長かったのに、全く別の世界だった
  • 同期が10歳若く、世代ギャップを感じた
  • 職場環境そのものが合わなかった

心理的な葛藤

39歳で新入社員。営業では先輩だったのに、ここでは圧倒的に不慣れ。「年寄りなのに新人扱い」という微妙な立場。全体像が見えない中で、何をしたらいいのか判断できない状態が続いた。

退職の決定

1年4ヶ月でSIer企業を退職。「逃げるようにして去った」という表現が、当時の心情を物語っている。

重要な学習

技術スキルだけではエンジニアではない

  • コミュニケーションスキルの重要性
  • 全体像把握の必要性
  • 職場環境・人間関係の適合性
  • 「今だったらもっとできる」という後発的な理解

結論

最初の現場は失敗だったが、「技術だけではダメ」という本質的な気づきが得られた

学習スタイル

環境の影響

コロナ禍での環境

会社の出勤者は少なく、ほぼ在宅勤務。自由度が高かった。会社から徒歩2分のところに引っ越し、365日仕事に取り組む環境が整った。

重要な気づき

「環境を変えるとめちゃめちゃ成長する」という実感。1社目での失敗を経て、2社目での自由度とサポート環境が、劇的なスキルアップを実現。3年10ヶ月の安定した環境があったからこそ。

結論

長期の安定環境 + 自由度 + 実務経験 = 本当のプロフェッショナルスキル形成

5プログラミング言語習得の本質

結論

最初の言語が最も大変。その後は言語の概念は同じ。最初の言語習得が全ての基礎

質問：どの技術習得が最も困難だったか？

セミナー参加者からの質問に対する回答。

ブーブーの回答

「Javaが一番大変だった」

Javaを習得した後、他の言語（Laravel、Reactなど）習得は比較的簡単。なぜか？

  • 最初の言語で「プログラミングの思想」を学ぶ
  • その後の言語習得は「文法の違い」を学ぶだけ
  • 基礎概念は全て共通。応用は容易

言語選択の重要性

最初の言語選びは極めて重要

難しい言語から始めるほど、後の学習が楽になる。Javaのような厳密な言語で基礎を学べば、Pythonなどの簡潔な言語は容易に習得できる。

結論

最初の言語習得 = 最大の投資。その後は複利効果で加速的にスキル習得が進む
